# Об исправлениях загрузки данных

## Проблема
При запуске приложения на главной странице выдавалась ошибка:
- "Ошибка загрузки категорий" (Unable to load listings)
- "Ошибка загрузки объявлений" (Unable to load listings)

## Причина
Код имел логирование либо закомментировано, либо отключено, что скрывало реальную причину ошибок. Разработчику было невозможно определить почему загрузка не работает.

## Решение
Включено детальное логирование всех ошибок API с:
1. **Полным текстом ошибки**
2. **Типом ошибки** 
3. **Stack trace**
4. **Формата ответа от API**

## Изменённые файлы

### 1. lib/blocs/listings/listings_bloc.dart
**Что изменилось**: Добавлено детальное логирование при ошибке загрузки

```dart
// Было:
catch (e) {
  emit(ListingsError(message: 'Unable to load listings'));
}

// Стало:
catch (e, stackTrace) {
  log.e(
    '❌ КРИТИЧЕСКАЯ ОШИБКА в LoadListingsEvent:\n'
    '   Сообщение: $errorMessage\n'
    '   Тип: ${e.runtimeType}\n'
    '   Ошибка: $e',
    error: e,
    stackTrace: stackTrace,
  );
  emit(ListingsError(message: 'Unable to load listings'));
}
```

### 2. lib/services/api_service.dart
**Что изменилось**: 
- Добавлено логирование в `getCatalogs()` 
- Добавлено логирование в `getAdverts()`

**Логирует**:
- Ключи ответа API
- Тип данных в поле 'data'
- Количество элементов в ответе
- Предупреждение если 'data' поле отсутствует

## Как использовать исправления

1. **Запустить приложение**:
   ```bash
   flutter run -v
   ```

2. **Посмотреть консоль для логов**:
   - При ошибке загрузки будет видно точное сообщение
   - Будут видны типы ошибок (сеть, парсинг, авторизация, etc.)

3. **Использовать DIAGNOSTICS.md для отладки**:
   - Проверить доступность API сервера
   - Вручную проверить ответ API через curl/Postman
   - Убедиться что формат JSON корректен

## Проверка работоспособности

Если логирование показывает:
```
✅ Успешно: ✅ API getCatalogs() response keys: [data, ...]
   - data type: List
   - data length: 8
```

То проблема **НЕ в LoadListingsEvent или getCatalogs()**.

Если видно ошибку типа:
```
❌ ОШИБКА при загрузке каталогов: Failed to load catalogs: ...
```

То нужно посмотреть полный текст ошибки в консоли и действовать согласно [DIAGNOSTICS.md](./api/DIAGNOSTICS.md).
